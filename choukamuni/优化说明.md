# GachaSimulator 优化说明（多文件拆分 + 配置面板 + 十连保底）

## 文件结构

| 文件 | 行数 | 职责 |
|------|------|------|
| `GachaSimulator.java` | ~350 | 主窗口、UI搭建、事件处理、抽卡逻辑 |
| `GameState.java` | ~20 | 所有可变抽卡状态（计数器、星声、保底标记） |
| `GachaItem.java` | ~30 | 单次抽卡结果数据（名称、稀有度、五星间距） |
| `GoldRecord.java` | ~10 | 五星记录数据（角色名、抽数） |
| `GachaResultRenderer.java` | ~70 | 结果列表按稀有度分色渲染 |
| `ProbabilityConfig.java` | ~15 | 可调整的概率参数（概率、保底抽数、消耗） |
| `SettingsDialog.java` | ~110 | 设置对话框（修改参数 → 保存生效） |

## 优化 A：拆分为多文件

### GachaSimulator.java

- **删除了** 4 个内部类定义（原第 554-563 行 `GameState`、原第 565-581 行 `GachaItem`、原第 583-588 行 `GoldRecord`、原第 115-163 行 `GachaResultRenderer`），各抽出为独立文件
- **新增第 23-24 行** — 引入 `config` 和 `state`（通过 new 实例化独立类）
- **第 40 行** — `fiveStarPool` 初始化中用 `state.upCharacter` 取代直接引用 `upCharacter`
- **第 42 行** — `pullCountLabel` 用 `state.totalPulls`
- **第 43 行** — `resourceLabel` 用 `state.xingShengCount`
- **全文** — 所有原本直接访问的散落字段改为 `state.xxx` 或 `config.xxx`

### GameState.java（新文件）

- **原有字段（原 GachaSimulator 第 29-31、46-49、54 行）**全部集中于此：
  ```java
  pullsSinceLastFiveStar, guaranteeNextLyn, upCharacter,
  pullsSinceLastFourStar, totalPulls, xingShengCount,
  infiniteMode, fiveStarRecords
  ```
- **构造函数**接收 `initialXingSheng` 参数，保证星声从配置初始化

### GachaItem.java（新文件）

- `updateLynTracking` 和 `addResult` 中原先直接访问 `item.name` / `item.rarity`，现改为 `item.getName()` / `item.getRarity()`

### GoldRecord.java（新文件）

### GachaResultRenderer.java（新文件）

  ```java
  costPerPull          = 160     // 原硬编码（第 462 行）
  ```

- **GachaSimulator.java 中的变更**：
  - 第 38-42 行 — 删除 `static final` 概率常量，改用 `config.xxx`
  - 第 292 行 — `onResetXingSheng` 中用 `config.initialXingSheng` 替代硬编码 16000
  - 第 330-332 行 — 十连资源检查中用 `config.costPerPull * 10` 替代硬编码 1600
  - 第 363 行 — 单抽资源检查中用 `config.costPerPull` 替代硬编码 160
  - 第 372 行 — 扣除星声用 `config.costPerPull`
  - 第 395 行 — 硬保底检查用 `config.hardPityPulls`
  - 第 405-408 行 — 软保底计算用 `config.softPityStart` 和 `config.softPityIncrement`
  - 第 412 行 — 四星概率用 `config.baseFourStarRate`
  - 第 407 行 — 五星概率用 `config.baseFiveStarRate`

  - 第 448 行 — `updatePityDisplay` 中五星保底显示用 `config.hardPityPulls`

### SettingsDialog.java（新文件，~110 行）

- **构造函数**：接收 `JFrame owner` 和 `ProbabilityConfig config`，弹出模态对话框
- **界面**：7 行 GridLayout 表单，每行"标签 + 文本框"
- **字段**：
  - 五星基础概率 (`baseFiveStarRate`)
  - 四星基础概率 (`baseFourStarRate`)
  - 软保底起始抽数 (`softPityStart`)
  - 软保底每抽递增 (`softPityIncrement`)
  - 硬保底抽数 (`hardPityPulls`)
  - 初始星声 (`initialXingSheng`)
  - 每抽消耗 (`costPerPull`)
- **保存按钮**：将文本框内容解析后写回 `config` 对象，标记 `confirmed = true`，关闭对话框
- **取消按钮**：直接关闭，不修改 config
- **格式校验**：`NumberFormatException` 捕获，弹出错误提示

### GachaSimulator.java 设置相关改动

- **第 188-189 行** — 新增"设置"按钮（灰色），点击打开 `SettingsDialog`
- **第 283-295 行** — 新增 `onOpenSettings()` 方法：
  ```java
  打开 SettingsDialog → 用户保存后:
    1. state.xingShengCount = config.initialXingSheng  // 重置星声
    2. 保底标签更新分母
    3. 调用 onResetPulls()  // 清空所有抽卡状态
    4. 刷新资源标签
  ```

## 优化 E：十连保底四星

### GachaSimulator.java

- **第 267 行** — `onDrawClick` 调用改为 `drawGacha(false)`（不强制四星）

- **第 271-288 行** — `onTenDrawClick` 重写为带保底逻辑：
  ```java
  boolean hasFourStarOrAbove = false;
  for (int i = 0; i < 10; i++) {
      boolean force = (i == 9 && !hasFourStarOrAbove);
      GachaItem item = drawGacha(force);
      if (item == null) break;
      if (!"三星".equals(item.getRarity())) {
          hasFourStarOrAbove = true;    // 标记已出四星以上
      }
      addResult(item);
  }
  ```
  - 前 9 抽正常抽，一旦出现四星或五星就标记 `hasFourStarOrAbove = true`
  - 第 10 抽如果仍没出过四星以上，传 `force=true` 强制升为四星

- **第 356-359 行** — `drawGacha(boolean forceFourStarOrAbove)` 新增参数：
  ```java
  String rarity = determineRarity();
  if (forceFourStarOrAbove && "三星".equals(rarity)) {
      rarity = "四星";  // 末抽保底：三星强制升四星
  }
  ```

- **第 366-382 行** — 资源不足检查改用 `config.costPerPull * 10`
